# 🤖 자율 에이전트 운영 지침 (AGENT.md)

이 문서는 자율 에이전트(Hermes 등)가 본 팟캐스트 저장소를 전담 관리하고 자율적으로 운영하기 위해 준수해야 하는 행동 강령 및 파이프라인 가이드입니다. 

에이전트는 인간의 상시 개입 없이 다음 수칙에 따라 하루 단위의 팟캐스트 생성을 완수해야 합니다.

---

## 1. 일일 운영 프로세스 (Workflow)

에이전트는 매일 정해진 스케줄러(cron 등)에 의해 기상한 후 다음 순서대로 작업을 전담 수행합니다.

### **[Step 1] 뉴스 정보 수집 및 정리 (Research)**
- 신뢰할 수 있는 IT/AI 뉴스 소스 또는 지정된 RSS/API를 탐색합니다.
- 오늘 날짜의 핵심 AI 뉴스를 요약하여 정리합니다.

### **[Step 2] 팟캐스트 에피소드 오디오 생성 (Synthesis)**
- 수집된 뉴스를 바탕으로 청취하기 좋은 친근한 팟캐스트 대본(Markdown)을 작성합니다.
- TTS(ElevenLabs, OpenAI Audio, 또는 NotebookLM MCP 등)를 호출하여 대본을 고품질 MP3 또는 M4A 오디오 파일로 합성합니다.
- 오디오 파일 이름 포맷: `AI_Daily_News_YYYY-MM-DD.mp3` (또는 `.m4a`)
- 저장 위치: `audio/` 디렉토리 아래에 저장합니다.

### **[Step 3] RSS 피드 업데이트 (Feed Update)**
- 에피소드 오디오 생성이 완료되면, 반드시 루트에 위치한 `config.yaml` 설정을 참조하여 `scripts/update_feed.py`를 실행합니다.
- **실행 명령어 예시**:
  ```bash
  python scripts/update_feed.py "[에피소드 제목]" "[오디오 파일명]" "[에피소드 상세 요약 설명]"
  ```
- 예시:
  ```bash
  python scripts/update_feed.py "2026년 AI는 프롬프트 없이 스스로 일한다" "AI_Daily_News_2026-06-01.mp3" "2026년 6월 1일 자 AI 데일리 뉴스 요약입니다."
  ```

### **[Step 4] 피드 규격 정합성 셀프 체크 (Self Validation - 필수)**
- 피드가 생성되거나 업데이트된 즉시, 에이전트는 피드의 XML 정법성 및 팟캐스트 필수 사양을 검증해야 합니다.
- **실행 명령어**:
  ```bash
  python scripts/validate_feed.py
  ```
- **오류 처리 (Troubleshooting)**:
  - 검증 단계에서 실패(Exit code 1)할 경우, 에이전트는 오류 로그를 분석하여 즉시 수정 루프를 수행합니다.
  - XML 문법 깨짐 시 `update_feed.py`를 재실행하거나, 누락된 `config.yaml` 요소를 확인하고 보완합니다.
  - **검증이 통과(Exit code 0)될 때까지 절대 5단계로 진입하지 마십시오.**

### **[Step 5] 커밋 및 배포 푸시 (Commit & Push)**
- 로컬 검증이 성공적으로 통과되면, 변경 사항을 Git에 스테이징하고 커밋하여 푸시합니다.
  ```bash
  git add audio/ feed.xml
  git commit -m "Add daily episode: YYYY-MM-DD"
  git push origin main
  ```
- 메인 브랜치에 푸시가 완료되면, GitHub Actions 배포 워크플로우가 자동으로 트리거되어 `gh-pages` 브랜치로 배포가 이루어지며, 팟캐스트 피드가 실시간 반영됩니다.

---

## 2. 에이전트 개발 규칙 및 행동 강령

1. **설정 파일(config.yaml) 준수**: 스크립트 내부에 메타데이터나 특정 URL을 하드코딩하지 마십시오. 오직 `config.yaml` 설정을 불러와 동적으로 바인딩해야 합니다.
2. **Git URL 자동 추론**: 에피소드 피드 빌드 시 `base_url`이 설정되어 있지 않은 경우, Git remote origin 정보를 통해 팟캐스트 서빙 주소를 유연하게 추론하여 삽입해야 합니다.
3. **무결성 사수**: `feed.xml`을 업데이트할 때, 기존 에피소드 아이템 히스토리가 절대 훼손되거나 유실되지 않도록 XML을 다뤄야 합니다.
4. **로깅 철저**: 모든 작업 성공 및 오류 분석 기록은 로컬 히스토리 또는 로깅 에이전트 채널을 통해 투명하게 기록하여 인간 사용자가 언제든 모니터링할 수 있도록 하십시오.

---

## 3. 배포 완료 후 유저 안내 지침 (Notification)

에이전트는 5단계(배포 푸시)를 완수하면, **인간 유저에게 작업 완료를 최종 보고할 때 반드시 아래 포맷으로 계산된 청취 링크를 명확히 안내**해야 합니다.

### **[유저 노티피케이션 권장 템플릿]**
```markdown
오늘 자 AI 팟캐스트 에피소드 발행이 완료되었습니다! 🎙️

- **오늘의 에피소드 제목**: [에피소드 제목]
- **청취용 개별 음성 파일 링크**: https://[owner_id].github.io/[repo_name]/audio/[오디오_파일명]
- **구독용 RSS 피드 주소 (팟캐스트 앱 등록용)**: https://[owner_id].github.io/[repo_name]/feed.xml

*(GitHub Pages 배포가 Actions에 의해 진행 중입니다. 페이지 반영 완료까지 약 30초~1분 정도 소요될 수 있으니 잠시 후 새로고침하여 청취해 주세요!)*
```

이로써 인간 유저가 주소를 직접 알아내거나 연산할 필요 없이, 에이전트 보고서 내에서 즉시 링크를 클릭하여 에피소드를 모니터링할 수 있도록 사용자 편의성을 보장하십시오.
