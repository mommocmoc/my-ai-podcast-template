# 🎙️ 나만의 AI 팟캐스트 방송국 만들기 (자동 배포 템플릿)

이 저장소는 **NotebookLM의 딥 리서치(Deep Research) 및 오디오 오버뷰(Audio Overview) 기능**을 활용하여, 자율 AI 에이전트가 매일 인터넷에서 핫한 최신 뉴스를 수집하고 라디오 방송 형태의 대화형 오디오를 제작해 **내 팟캐스트 사이트에 자동으로 올릴 수 있도록 돕는 초간편 템플릿**입니다.

비개발자분들도 코딩 에이전트(Hermes 등)에게 프롬프트 한 줄만 던지면, 3분 만에 나만의 데일리 AI 방송국을 개설할 수 있습니다!

---

## 🚀 3분 만에 시작하는 초간편 개설 가이드

어려운 코딩이나 명령어 입력은 에이전트가 알아서 수행합니다. 아래 3단계만 따라 하세요!

### 1단계: 내 저장소로 복제하기
1. 화면 우측 상단의 **`Use this template`** 버튼을 클릭하여 본인의 GitHub 계정으로 새 저장소를 복제(Fork)해 만듭니다.

### 2단계: 에이전트에게 통째로 위임하기
1. 로컬에 저장소를 가져온 후, 코딩 에이전트(Hermes, Goose 등)를 켭니다.
2. 에이전트 대화창에 아래 **마법의 프롬프트**를 그대로 복사해서 입력해 줍니다.
   > **💬 마법의 위임 프롬프트:**
   > *"안녕! 이 팟캐스트 저장소를 나와 대화하면서 처음부터 끝까지 다 세팅해줘. `config.yaml` 설정 파일 작성부터 구글 NotebookLM 연동까지 단계별로 아주 쉽고 친절하게 알려주고, 세팅이 끝나면 매일 아침 뉴스를 큐레이션해서 방송을 자동 배포하는 스케줄러(크론 잡)까지 구성해서 기동해줘."*

### 3단계: 발행된 내 팟캐스트 링크로 즉시 청취하기
1. 에이전트가 세팅 및 첫 에피소드 발행을 마친 뒤, 알려주는 청취 링크를 눌러서 즉시 방송을 감상하세요!
2. 내 팟캐스트 구독용 RSS 피드 링크를 스마트폰 팟캐스트 앱(Apple Podcasts 등)에 등록하면 언제 어디서나 구독해서 들을 수 있습니다.

## 💬 에이전트와의 친절한 협업 흐름 (티키타카 시나리오)

에이전트에게 위임하면, 에이전트가 다음과 같이 친절하게 말을 걸며 단계별로 세팅을 완료해 줍니다.

* **[대화 1] 팟캐스트 기본 정보 적기**
  > 🤖 **에이전트**: *"안녕하세요! 나만의 팟캐스트 개설을 환영합니다. 방송의 제목과 저자 이름, 연락처 이메일을 무엇으로 하고 싶으신가요?"*
  > 👤 **유저**: *"제목은 '김철수의 매일 AI 핵심 요약', 저자는 '김철수', 이메일은 'chulsoo@example.com'으로 해줘!"*
  > 🤖 **에이전트**: *"알겠습니다! `config.yaml` 파일에 해당 정보를 완벽하게 채워 넣었습니다."*

* **[대화 2] NotebookLM 오디오 생성 로그인 돕기 (1회 인증)**
  > 🤖 **에이전트**: *"이제 고품질 음성을 생성하는 구글 NotebookLM에 로그인해 볼까요? 보안상 로그인 정보는 한 번 복사해 주셔야 합니다.*
  > *브라우저에서 [NotebookLM](https://notebooklm.google.com)에 로그인하신 다음, F12 개발자 도구의 'Application -> Cookies'에서 세션 쿠키를 복사해 대화창에 알려주세요. 제가 로그인 세팅을 직접 마무리하겠습니다!"*
  > 👤 **유저**: *(쿠키 값을 복사해 전송)*
  > 🤖 **에이전트**: *"감사합니다! NotebookLM 자동 로그인 세팅을 무사히 완료했습니다."*

* **[대화 3] 큐레이션 주제 선정하기**
  > 🤖 **에이전트**: *"매일 어떤 사이트의 어떤 뉴스를 큐레이션해서 요약 방송을 제작해 드릴까요?"*
  > 👤 **유저**: *"매일 아침 'Hacker News'에서 가장 인기 있는 인공지능(AI) 뉴스를 찾아서 정리해줘."*
  > 🤖 **에이전트**: *"좋습니다! 뉴스 큐레이션 크론 스케줄러를 등록했습니다. 지금 즉시 첫 방송을 빌드하고 발행해 드리겠습니다."*

## 🔗 내 팟캐스트 주소 확인하는 법

모든 배포는 깃허브 서버에서 자동으로 이뤄집니다. 유저가 깃허브 웹 화면의 복잡한 세팅창에 들어가지 않아도, 에피소드가 푸시되는 즉시 아래의 주소로 자동 발행됩니다.

* **구독용 RSS 피드 주소** (Apple Podcasts, Spotify 등 등록용):
  > `https://<내_GitHub_아이디>.github.io/<저장소_이름>/feed.xml`
* **에피소드 음성 파일 직접 재생 주소**:
  > `https://<내_GitHub_아이디>.github.io/<저장소_이름>/audio/<음성_파일명.mp3>`
* **팟캐스트 대표 커버 이미지 주소**:
  > `https://<내_GitHub_아이디>.github.io/<저장소_이름>/cover.png`

*(예: 깃허브 아이디가 `chulsoo`이고 저장소 이름이 `my-podcast`라면, 내 팟캐스트 구독 주소는 `https://chulsoo.github.io/my-podcast/feed.xml`이 됩니다.)*

### 1. 로컬 가상 환경 구동
```bash
pip install -r requirements.txt
```

### 2. 수동 피드 업데이트 명령어
```bash
python scripts/update_feed.py "에피소드 제목" "파일명.mp3" "상세 설명 요약"
```

### 3. 피드 표준 정합성 검사
```bash
python scripts/validate_feed.py
```

### 4. 팟캐스트 피드 포맷팅 정돈
```bash
npm install
npm run format
```

### 5. 배포 작동 원리
본 템플릿의 `.github/workflows/deploy_pages.yml` 파일은 `main` 브랜치에 푸시가 발생할 때마다 작동합니다. 에셋들이 수합되면 배포용 전용 브랜치(`gh-pages`)로 자동 전송하며, 이를 통해 깃허브가 Pages 서비스를 자동으로 활성화하여 정적 호스팅을 지원합니다.
(Actions가 원활하게 push를 하려면 저장소의 `Settings` -> `Actions` -> `General` -> `Workflow permissions`에서 `Read and write permissions`가 반드시 켜져 있어야 합니다.)
